# YouTube API Developer Policies 요약 메모

## 1. 문서 정보

- 출처: Google for Developers, `YouTube API Services - Developer Policies`
- 보조 출처: Google for Developers, `YouTube Data API overview`
- URL: https://developers.google.com/youtube/terms/developer-policies?hl=ko
- 확인일: `2026-03-13`
- 용도: 박사학위논문 IRB 제출용 연구계획서와 자료관리 문구를 외부 플랫폼 정책에 맞게 보수적으로 조정하기 위한 작업 메모

이 문서는 원문을 그대로 옮긴 스크랩이 아니라, IRB 문맥에 직접 영향을 주는 정책 항목만 재서술한 요약이다. 실제 제출 직전과 실제 자료 수집 직전에는 반드시 원문 최신본을 다시 확인해야 한다.

## 2. IRB 초안에 직접 영향을 주는 핵심 정책

### 2-1. 공식 API만 사용해야 함

- 자료 수집은 유튜브가 제공하는 공식 API와 허용된 기능 범위 안에서 수행해야 한다.
- 비공식 스크래핑, 우회 접근, 제한 회피 방식의 수집은 쓰지 않는 것이 안전하다.
- 따라서 IRB 초안에는 `공식 API 이용`, `비공식 스크래핑 금지`를 명시하는 것이 적절하다.

### 2-2. 필요한 범위만 수집해야 함

- 정책 문맥상 API 데이터는 허용된 목적과 기능 범위 안에서 최소한으로 다뤄야 한다.
- 연구에는 영상·댓글·채널의 공개 메타데이터와 공개 텍스트만 사용하고, 연구문제와 무관한 항목은 넓게 저장하지 않는 쪽이 안전하다.

### 2-3. 일일 API 할당량 범위 안에서 반복 수집해야 함

- YouTube Data API는 프로젝트별 일일 할당량 체계 안에서 운영된다.
- 따라서 IRB 문서에는 `허용된 일일 API 할당량 범위 안에서 반복 수집`, `수집 범위에 해당하는 공개 영상과 해당 댓글 수집이 완료될 때까지 순차 진행`이라는 운영 문구를 두는 편이 안전하다.

### 2-4. 시청각 원본 파일 다운로드·저장은 피해야 함

- 정책상 시청각 콘텐츠를 다운로드, 저장, 백업, 캐싱하는 행위는 매우 제한적으로 취급된다.
- 따라서 본 연구에서는 영상 원본이나 음성 원본을 내려받아 별도 보관하지 않는 방향이 안전하다.
- 텍스트 분석에는 공개 자막 또는 공개 전사 텍스트가 있는 경우만 활용하고, 없는 경우에는 메타데이터 수준 분석 또는 제외 처리로 정리하는 것이 적절하다.

### 2-5. 승인되지 않은 API 원자료 및 원문 텍스트의 장기 보관은 주의가 필요함

- 정책에는 승인되지 않은 API 데이터의 임시 저장 기간 제한과 삭제 또는 갱신 의무가 포함되어 있다.
- 현재 연구처럼 연구참여자의 개별 OAuth 승인 없이 공개 자료를 수집하는 경우, raw API 응답, 식별 가능한 원자료, 댓글 원문과 같은 row-level 텍스트 자료를 장기 보관하는 문구는 정책과 충돌할 수 있다.
- 따라서 IRB 초안에서는 `원자료 장기 보관` 대신 `비식별 분석산출물 장기 보관`, `원자료는 최종 수집일 이후 30일 이내 삭제`로 정리하는 것이 보수적이다.

### 2-6. 식별 가능한 사용자 수준의 민감 속성 추론은 보수적으로 다뤄야 함

- 정책 취지상 API 데이터를 이용해 식별 가능한 개인의 민감한 속성을 추론, 저장, 추적하는 방식은 매우 조심해야 한다.
- 본 연구의 `정서/입장 분석`은 특정 사용자의 정치적 성향이나 민감 정보를 판정하는 작업이 아니라, 댓글과 담론의 반응 구조를 집계 수준에서 해석하는 절차라는 점을 명확히 적는 것이 안전하다.

### 2-7. 결과 공개와 재배포는 제한적으로 해야 함

- 원자료 재배포, 검색 가능한 원문 노출, 식별자 재노출은 정책 및 IRB 위험관리 모두에 불리하다.
- 따라서 논문과 발표자료에는 직접 인용, 스크린샷, URL, account/channel ID, 채널명 등을 원칙적으로 싣지 않는 방향이 적절하다.

## 3. 현재 IRB 문서에 반영한 조정 사항

- 자료 수집은 `유튜브 공식 API`만 사용하고, 비공식 스크래핑은 하지 않는다고 명시했다.
- 자료 수집은 `IRB 승인 후` 가능한 시점부터 시작하고, 일일 API 할당량 범위 안에서 반복 수행한다고 명시했다.
- 영상 원본 또는 음성 원본을 내려받아 자동 전사하는 절차는 빼고, `공개 자막/공개 전사 텍스트만 사용`하는 쪽으로 수정했다.
- `연구자료 5년 보관` 문구는 유지하되, 그 대상을 `직접 식별자와 원문 댓글을 제거한 분석산출물, 코드북, 집계표, 통계·네트워크 지표, 재현성 문서` 중심으로 좁혔다.
- 승인되지 않은 API 원자료, 식별 가능한 원자료, 원문 댓글은 `정책상 허용 범위를 넘겨 장기 보관하지 않음`, `최종 수집일 이후 30일 이내 삭제` 원칙을 추가했다.
- `정서 및 입장 분석`은 식별 가능한 개인의 정치적 성향 등 민감 속성 판정이 아니라 담론 단위 분석이라는 점을 적었다.
- `직접 인용·스크린샷 원칙적 미사용` 방침을 유지하고, YouTube 정책 준수와 연결되는 근거로도 정리했다.
- 실무 운영안은 `IRB 승인 후 ~ 최종 수집일`의 고정 수집 창을 설정하고, 그 기간에는 일일 API 할당량 범위 안에서 자료 수집을 반복하며, 그 이후 추가 refresh 또는 update 없이 확정 코퍼스를 분석하는 방식으로 정리했다.

## 4. 아직 사용자 확인이 필요한 쟁점

- 실제 수집 파이프라인이 `최종 수집일 + 30일` 이내 원자료 삭제와 분석산출물 확정을 끝낼 수 있는지
- 장기 보관할 분석자료에 원문 댓글 전문이나 raw API 응답이 남지 않도록 설계되어 있는지
- 공개 자막이 없는 영상을 텍스트 분석에서 제외해도 연구문제상 문제가 없는지
- 실제 제출 시점에도 동일 정책 문구가 유지되는지
