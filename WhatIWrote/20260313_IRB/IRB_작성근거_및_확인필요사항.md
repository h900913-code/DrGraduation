# IRB 초안 작성 근거 및 확인 필요 사항

## 0. 이번 사용자 지시로 확정한 사항

다음 사항은 이번 사용자 지시로 확정하여 초안에 반영하였다.

- IRB 제출 경로는 `일반심의`로 간다.
- `심의면제 불가`를 전제로 작성한다.
- 연구책임자는 `박정현`, 지도교수는 `윤순진`으로 확정한다.
- 검색어는 `기후변화` 단일 규칙으로 고정한다.
- 연구 수행 일정의 시작 시점은 `2026-02-10`으로 잡는다.
- 본 연구용 실제 자료 수집 시작일은 고정 날짜가 아니라 `IRB 승인 후`로 둔다.
- 본 연구용 자료 수집은 `최종 수집일`까지 수행한 뒤 추가 refresh 또는 update 없이 코퍼스를 확정한다.
- IRB 문서상 자료 수집 및 분석 범위는 `2015-10-12 ~ 2026-03-27` 전체로 둔다.
- 분석대상은 기본 검색어 `기후변화`로 검색된 공개 영상 전체와 그 영상들에 달린 공개 댓글 및 공개 대댓글 전체로 정리한다.
- 자료 수집은 일일 API 할당량 범위 안에서 반복 수행하는 방식으로 정리한다.
- 승인되지 않은 API 원자료, 식별 가능한 원자료, 원문 댓글은 `최종 수집일`로부터 `30일` 이내 삭제한다.
- 장기 보관 대상은 원문 복원이 어려운 비복원적 분석산출물로 한정한다.
- 유니크 ID 기준 안전상 상한은 `1,500,000 public YouTube account/channel IDs`로 유지하되, 중복 제거 없는 `comments` 행 기준 안전한 상단 planning 값은 `1,800,000 comments`로 병기한다.
- IRB 시스템이 단일 수치만 입력받는 경우에는 입력 편의상 더 보수적인 값인 `2,000,000`을 권고한다.
- 대상군 산정의 기준 시점은 `2026-03-13` 내부 메모 기준으로 간다.
- 동의 관련 문서는 `서면동의면제사유서`로 준비한다.

이에 따라 기존 초안의 `심의면제 가능성` 관련 유보 문구는 정리하고, 일반심의 제출용 논리로 재작성하였다.

## 4. 최신본과 이전 문서의 충돌 여부

### 실질적 충돌이 없는 부분

- 연구 주제
- 핵심 자료원: 공개 YouTube 영상, 댓글, 채널/계정 단위 자료
- 분석 축: 토픽, 정서·태도, 네트워크
- 연구윤리 인식: 비식별화, 재식별 위험 통제, 인용 최소화

### 부분적으로 변화한 부분

- 범위는 비교 가능성 탐색에서 `한국 단일권역` 중심으로 더 명확해졌으나, IRB 문서에서는 검색 규칙 기반의 재현 가능한 수집 기준으로 다시 정리되었다.
- IRB 제출용 문맥에서는 자료 수집 및 분석 범위를 `2015-10-12 ~ 2026-03-27` 전체로 넓히고, 그 범위 전체를 분석 대상으로 읽는 완충 구간 논리가 추가되었다.
- IRB 경로에 대해서는 초기에는 `문의/판단 필요`가 강조되었고, 2026-03-12 이후 내부 메모와 전화 요약에서는 `general new review` 준비 방향이 더 강해졌다.
- 검색어 운용은 최신본에서 기본 검색어 `기후변화` 중심으로 더 단단해졌고, 관련 표현은 보조 검토 대상으로 남았다.

### 정리

연구의 본질적 설계와 자료원은 충돌하지 않는다. 오히려 시간이 지나면서 공개자료 이용, 윤리 통제 원칙, 검색 규칙 기반 수집 기준이 더 명료해졌다. 따라서 초안 작성 시에는 최신 `Full_KOR`을 따르고, 이전 문서는 IRB 문장 보강용 보조 근거로만 활용하였다.

추가로 `YouTube API Services Developer Policies`를 확인한 결과, 원자료 장기보관 및 시청각 자료 취급 방식에 외부 플랫폼 정책상 제약이 존재함을 확인하였다. 이에 따라 IRB 초안에서는 `공식 API만 사용`, `시청각 원본 비저장`, `승인되지 않은 API 원자료·식별 가능한 원자료·원문 댓글의 단기 보관 및 30일 내 삭제 원칙`, `민감 속성 프로파일링으로 읽히지 않도록 분석 서술 보정`을 반영하였다.

## 5. 항목별 확실한 내용과 불확실한 내용

### 5-1. 근거가 명확한 항목

- 연구 제목과 주제
- 연구 목적과 배경
- 연구문제 3개
- 자료원: 공개 YouTube 영상, 공개 댓글, 공개 대댓글, 공개 채널 메타데이터
- 자료 수집 방식: 유튜브 공식 API와 공개 검색
- 자료 수집 범위: `2015-10-12` ~ `2026-03-27`
- 자료 수집 범위 전체를 분석 대상으로 읽는 원칙
- 분석 절차: 기초통계, 토픽 분석, 정서·태도 분석, 네트워크 분석
- 윤리 원칙의 핵심: 가명처리, 재식별 위험 통제, 원문 인용 최소화, 공개 범위 제한
- 검색어 규칙: `기후변화` 단일 규칙
- 연구대상 수 산정 방식: `public YouTube account/channel IDs`의 유니크 수와 중복 제거 없는 `videos/comments` 행 수를 병기
- 유니크 ID 기준 중심 추정 참고치: `1,000,000`
- 유니크 ID 기준 안전상 상한: `1,500,000`
- comments 행 기준 중심 추정 참고치: `1,635,469`
- comments 행 기준 안전한 상단 planning 값: `1,800,000`
- videos 행 기준 중심 추정 참고치: `39,128`
- videos 행 기준 안전한 상단 planning 값: `42,000`
- 단일 입력값 권고안: `2,000,000`

  - 반영안: 메인 초안에서는 `유니크 ID 기준`과 `행 기준`을 병기하고, 단일 입력값은 `2,000,000`을 권고하는 방향으로 정리

  - 근거: 가이드에는 `동의서 3년`, `연구자료 5년`, 식별정보 분리 원칙이 있음
  - 불확실성 원인: 실제 저장장치, 암호화 방식, 접근자 명단, raw key 파일 파기 시점이 최신본에 명시되지 않음

  - 사용자 결정: 대상군 산정은 `2026-03-13` 기준 메모를 따름
  - 반영 방식: 논문 분석기간과 별개로, IRB의 분석대상군 규모 설명은 `2026-03-13` 기준 내부 메모에 따른 관리상 수치로 기재

  - 근거: 최신본은 원문 인용 최소화를 말하고, 이전 IRB 검토 메모는 검색 가능성 위험을 지적함
  - 제안안: 직접 인용과 스크린샷은 원칙적으로 하지 않는 것으로 쓰고, 불가피한 경우만 별도 지도교수 검토 아래 제한적으로 허용

  - 근거: 공식 `Developer Policies`에서 비공식 스크래핑 금지, 시청각 콘텐츠 다운로드·저장 제한, 승인되지 않은 API 원자료의 `30일` 보관 제한을 확인
  - 반영안: 공식 API만 사용, 시청각 원본 비저장, `IRB 승인 후 ~ 최종 수집일`의 고정 수집 창 설정, 일일 API 할당량 범위 내 반복 수집, 승인되지 않은 API 원자료·식별 가능한 원자료·원문 댓글의 `30일` 내 삭제, 장기 보관은 비복원적 분석산출물 중심으로 전환

  - 사용자 결정: 검색어는 `기후변화` 단일 규칙으로 고정

### 5-3. 자료 부족으로 [확인 필요] 처리한 항목

- 실제 자료 수집 파이프라인이 `최종 수집일 + 30일` 이내에 원자료 삭제와 비복원적 분석산출물 확정을 끝낼 수 있는지
- 장기 보관 예정 자료가 원문 댓글 또는 raw API 응답을 포함하지 않고도 재현성 확보에 필요한 수준을 충족하는지

다만 아래 항목은 이번 수정에서 확정하거나 `제안안`을 넣어 초안 본문에 반영했다.

- 연구책임자는 `박정현`으로 기재
- 지도교수는 `윤순진`으로 기재하고, `공동연구자 1인`으로 정리
- 원자료 접근자는 `박정현 + 윤순진`으로 최소화
- 개인정보관리책임자는 `박정현`으로 설정
- 보관 위치는 `비밀번호가 설정된 연구용 개인 컴퓨터 내 암호화 폴더 + 암호화된 백업 저장공간`으로 제안
- 기존 내부 DB는 `대상군 규모 산정과 자료구조 점검을 위한 예비 구축 자료`로 설명
- 연구일정은 `2026-02-10 ~ 2027-02-28` 범위의 일반심의 제출 및 분석 일정안으로 조정
- 검색어는 `기후변화` 단일 규칙으로 확정
- YouTube API 정책을 고려하여 시청각 원본 다운로드·저장은 하지 않고, 공개 자막·공개 전사 텍스트만 활용하는 안으로 조정
- 본 연구용 실제 자료 수집 시작일은 `IRB 승인 후`로 두고, `최종 수집일` 이후 추가 refresh 또는 update 없이 코퍼스를 확정하는 안으로 조정
- 승인되지 않은 API 원자료, 식별 가능한 원자료, 원문 댓글은 `최종 수집일`로부터 `30일` 이내 삭제하는 안으로 조정
- API 원자료 장기보관 대신 비복원적 분석산출물, 코드북, 통계·네트워크 지표 중심 보관안으로 조정

## 6. 연구참여자의 선정 기준 항목에 반영한 판단

`연구참여자의 선정 기준`과 `목표 연구참여자의 수 및 산출 근거`는 전통적 인간대상연구 문법과 현재 연구설계가 가장 강하게 충돌하는 지점이다. 이번 초안에서는 다음 판단을 적용했다.

- `연구참여자`를 직접 모집된 자연인으로 쓰지 않고, `공개 자료의 분석대상`이라는 방식으로 재해석했다.
- 분석대상 규모는 `public YouTube account/channel IDs`의 유니크 수와 중복 제거 없는 `videos/comments` 행 수를 병기했다.
- 유니크 ID 기준 상한은 `1,500,000`, comments 행 기준 상단 planning 값은 `1,800,000`으로 적었다.
- 단일 입력란에는 입력 편의상 더 보수적인 값인 `2,000,000`을 권고하는 방향으로 정리했다.
- `1,500,000`은 유니크 ID 기준 관리상 상한이고, `1,800,000`은 comments 행 기준 planning 상한이며, `2,000,000`은 행정 입력용 보수적 단일 수치라는 점을 명시했다.
- `ID 수는 자연인 수와 동일하지 않다`는 점을 본문에 넣었다.
- `comments` 행 수는 동일인의 반복 댓글을 포함한 총행 수라는 점을 본문에 넣었다.

이 판단은 `WhatIWrote\20260313 irb_target_population_estimate.md`와 `WhatIWrote\20260313_IRB\IRB_분석행규모_추정_보충메모.md`의 수치를 함께 반영한 것이다.

- 정서·입장 분석은 식별 가능한 개인의 정치적 성향 등 민감 속성을 판정하는 연구가 아니라 댓글·담론 단위 분석이라는 점을 명시했다.
- 기존 자료 보관 위치나 공동연구자 명단처럼 근거가 없는 운영 정보를 지어내지 않고 `[확인 필요]`로 남겼다.

- IRB 입력 화면에서 단일 숫자 입력란에 `2,000,000`을 넣고, 본문에서 `1,500,000 IDs / 1,800,000 comments`를 병기하는 방식이 제출 실무와 맞는지
