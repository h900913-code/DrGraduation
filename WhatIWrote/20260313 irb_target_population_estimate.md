# IRB 연구대상군 예상수치 산정 메모

작성일: 2026-03-13  
기준 DB: `C:\Repositories\Youtube Crawling\data\youtube_crawler_unified.db`

## 1. 목적

이 문서는 IRB 제출용 연구대상군 예상수치를 산정한 과정을 정리한 메모이다.  
여기서의 "대상군 수"는 자연인 수가 아니라, DB 상에서 식별 가능한 **공개 YouTube 계정/채널 ID의 유니크 수**를 의미한다.

- 업로더: `videos.channel_id`
- 댓글 작성자: `comments.author_channel_id`
- 최종 대상군 수: 두 집합의 **합집합(UNION)** 기준

주의:

- YouTube 계정/채널 ID는 자연인 1명과 1:1 대응한다고 보장할 수 없다.
- 따라서 IRB 문구에는 가능하면 "public YouTube account/channel IDs" 또는 "공개 유튜브 계정/채널 ID"라고 쓰는 것이 안전하다.

## 3. 현재 검색 커버리지

현재 DB의 완료된 검색 구간은 연속 1개가 아니라 2개 구간이다.

- 완료 구간 1: `2015-12-12 09:00:00 KST` 이상 `2022-01-01 09:00:00 KST` 미만
- 완료 구간 2: `2026-01-31 01:25:55 KST` 이상 `2026-03-05 01:26:55 KST` 미만
- 부분 검색 구간: `2022-01-01 09:00:00 KST` 이상 `2022-02-01 09:00:00 KST` 미만
- 큰 미검색 구간: `2022-02-01 09:00:00 KST` 이상 `2026-01-31 01:25:55 KST` 미만

기간 길이로 보면 완료 구간은 전체 의도 기간의 약 60.0%이다.

- 전체 의도 기간: `2015-12-12` ~ `2026-03-13`
- 완료 기간 비율: 약 `0.5996`

참고로 모든 `search.list` 호출 조건은 아래로 고정되어 있었다.

## 4. 댓글 작성자 수 추정 로직

### 4.1 기준 앵커

미검색 구간의 월별 댓글 작성자 수를 추정하기 위해 아래 두 관측치를 앵커로 사용했다.

- `2022-01`: 유니크 댓글 작성자 `4,770`
- `2026-02`: 유니크 댓글 작성자 `28,634`

중요한 점:

- `2022-01`은 아직 미완료 월이다.
- 해당 월 search window는 `pending`, `page_count_processed=4`, `resume_page_token`이 남아 있다.
- 따라서 `4,770`은 **하한(lower bound)에 가까운 값**으로 보는 것이 맞다.

### 4.2 월간 중복 보정

월별 유니크 댓글 작성자 수를 그대로 더하면, 여러 달에 반복 등장한 계정이 중복 계산된다.  
이를 보정하기 위해 "월별 합 대비 최종 유니크 비율"을 참고했다.

관측된 비율:

- `2019-2021`: `152,872 / 174,181 = 0.8777`
- `2020-2021`: `92,740 / 107,414 = 0.8634`
- `2026 recent`: `30,796 / 30,950 = 0.9950`

더 타이트한 추정을 위해, 과거 장기구간보다는 최근 급증 구간을 일부 반영한 `0.92`를 사용했다.

```text
추가 댓글작성자 수 = (미검색 48개월의 월별 유니크 댓글작성자 합) x 0.92
```

### 4.3 시나리오

미검색 48개월(`2022-02` ~ `2026-01`)의 월별 유니크 댓글 작성자 수를 세 가지 방식으로 보간했다.

1. `Geometric`: 완만한 기하 성장
2. `Hybrid`: 선형과 기하의 절충
3. `Linear`: 강한 선형 성장

결과:

| 시나리오 | 미검색 48개월 월별 합 | 중복 보정 후 추가 댓글작성자 | 최종 댓글작성자 총수 |
| --- | ---: | ---: | ---: |
| Geometric | 635,812 | 584,947 | 886,083 |
| Hybrid | 718,754 | 661,253 | 962,389 |
| Linear | 801,696 | 737,560 | 1,038,696 |

여기서 `최종 댓글작성자 총수`는 아래 식으로 계산했다.

```text
최종 댓글작성자 총수 = 현재 댓글작성자 301,136 + 추가 댓글작성자
```

## 5. 업로더 채널 수 추정 로직

업로더 채널은 댓글 작성자보다 규모가 작지만, 최종 연구대상군 합집합에는 포함되어야 한다.

### 5.1 기준 앵커

- `2022-01`: 유니크 업로더 `140`
- `2026-02`: 유니크 업로더 `711`

### 5.2 월간 중복 보정

업로더는 월간 재등장 비율이 댓글 작성자보다 높다. 관측된 비율은 아래와 같다.

- `2019-2021`: `2,578 / 4,302 = 0.5993`
- `2020-2021`: `2,074 / 3,316 = 0.6255`
- `2026 recent`: `763 / 794 = 0.9610`

더 타이트한 상향 추정을 위해 `0.70`을 사용했다.

```text
추가 업로더 수 = (미검색 48개월의 월별 유니크 업로더 합) x 0.70
```

### 5.3 댓글작성자와의 중복 보정

추가 업로더를 그대로 합집합에 더하면 댓글 작성자와 중복되는 계정이 과대계상된다.  
현재 및 연도별 관측치를 참고해 업로더 추가분의 `15%`를 댓글 작성자와 중복되는 것으로 보정했다.

관측치:

- 현재 전체 중복률: `762 / 4,929 = 15.5%`
- 연도별 중복률:
  - `2019`: `88 / 691 = 12.7%`
  - `2020`: `150 / 933 = 16.1%`
  - `2021`: `248 / 1,401 = 17.7%`
  - `2026`: `115 / 763 = 15.1%`

따라서:

```text
합집합에 순증하는 업로더 수 = 추가 업로더 수 x 0.85
```

## 6. 최종 유니크 대상군 추정

최종 연구대상군 수는 아래 식으로 계산했다.

```text

= 현재 유니크 합집합
+ 추가 댓글작성자
```

즉:


## 7. 150만 명 가정의 의미

`1,500,000` 유니크 계정/채널을 상정하려면, 현재 합집합 `305,303`에서 추가로 아래 수치가 더 필요하다.

```text
추가 필요 합집합 = 1,500,000 - 305,303 = 1,194,697
```

이를 미검색 48개월에 나누면:

```text
월평균 순증 유니크 합집합 필요치 = 1,194,697 / 48 = 24,889.5
```

즉 `1.5M`은 아래와 같은 의미를 가진다.

- 거의 모든 미검색 월이 `2026-02`에 가까운 고밀도 패턴을 지속해야 함
- 완전히 불가능한 값은 아님
- 그러나 현재 모델링 결과의 **중심 추정치보다 상당히 높은 상단 시나리오**임

참고로, 매우 공격적으로 `2026-02`급 월이 거의 계속된다고 가정하면 `1.59M` 수준도 산술상 가능하다.  
하지만 그 경우는 중앙값 추정이 아니라 "넓은 행정상 상한"에 가깝다.

## 8. IRB 보고용 권고

### 8.1 한 숫자만 써야 하는 경우

가장 무난한 단일 예상치는 아래를 권고한다.

- `1,000,000 public YouTube account/channel IDs`

이 숫자는:

- 현재 DB 관측치보다 충분히 큰 값이고
- Hybrid/Linear 사이 중앙 범위와도 부합하며
- `1.5M`처럼 과도하게 높은 상단값까지 가지는 않기 때문이다

### 8.2 예상치와 상한을 같이 쓸 수 있는 경우

아래 표현을 권고한다.

- 예상 연구대상군: 약 `1,000,000`
- 보수적 상한: 약 `1,200,000`
- 매우 넓은 안전상 상한: 약 `1,500,000`

해석:

- `1.0M`은 중심 추정에 해당
- `1.2M`은 IRB 계획상 보수적 상단으로 무난
- `1.5M`은 "충분히 넓게 잡은 관리상 상한"으로는 사용 가능하나, 중심 추정치로 보기에는 큼

## 9. 재현용 핵심 SQL

아래 SQL은 현재 DB 관측치를 다시 확인할 때 사용할 수 있다.

### 9.1 현재 업로더/댓글작성자/합집합

```sql
SELECT COUNT(DISTINCT channel_id) AS uploaders
FROM videos
WHERE channel_id IS NOT NULL AND channel_id <> '';

SELECT COUNT(DISTINCT author_channel_id) AS commenters
FROM comments
WHERE author_channel_id IS NOT NULL AND author_channel_id <> '';

SELECT COUNT(*) AS overlap
FROM (
    SELECT DISTINCT channel_id AS id
    FROM videos
    WHERE channel_id IS NOT NULL AND channel_id <> ''
    INTERSECT
    SELECT DISTINCT author_channel_id AS id
    FROM comments
    WHERE author_channel_id IS NOT NULL AND author_channel_id <> ''
);

SELECT COUNT(*) AS union_accounts
FROM (
    SELECT DISTINCT channel_id AS id
    FROM videos
    WHERE channel_id IS NOT NULL AND channel_id <> ''
    UNION
    SELECT DISTINCT author_channel_id AS id
    FROM comments
    WHERE author_channel_id IS NOT NULL AND author_channel_id <> ''
);
```

### 9.2 검색 커버리지

```sql
SELECT status, COUNT(*) AS cnt
FROM search_windows
GROUP BY status
ORDER BY status;

SELECT MIN(start_utc) AS min_start, MAX(end_utc) AS max_end
FROM search_windows;

SELECT MIN(published_after) AS min_after, MAX(published_before) AS max_before
FROM request_log
WHERE endpoint = 'search.list' AND success = 1;
```

## 10. 최종 요약

현재 DB와 성장 가정을 종합하면:

- 모델 기반 중심 추정: 약 `0.98M`
- 현실적 범위: 약 `0.90M ~ 1.06M`
- IRB 단일 예상치 권고: `1.0M`
- IRB 상한(cap) 권고: `1.2M`
- 아주 넓은 행정상 안전상 상한: `1.5M`

따라서 IRB 문서에서는 아래 표현이 가장 방어적이다.

> 본 연구의 분석 단위는 자연인이 아니라 공개 YouTube account/channel ID이며, 현재 수집분과 미수집 구간에 대한 보수적 추정치를 합산할 때 예상 연구대상 규모는 약 1,000,000 account/channel IDs, 넓은 관리상 상한은 약 1,500,000 IDs로 산정하였다.
